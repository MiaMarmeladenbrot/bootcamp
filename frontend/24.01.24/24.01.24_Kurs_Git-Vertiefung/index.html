GitHub Cheatsheet

DER Onlinedienst zur Softwareentwicklung und Versionsverwaltung für Softwareprojekte auf Git-Basis.

Repository erstellen

  1. in Github auf das Plus Icon klicken
  2. Owner, Title und Description hinzufügen
  3. private oder public einstellen
  4. Create repository
  5. Code kopieren
    • git init ⇒ initialisiert ein neues Git Repository im aktuellen Verzeichnis
    • git add README.md ⇒ optional; fügt eine readme-Datei hinzu
    • git commit -m "first commit" ⇒ erstellt ein neuen Commit mit dem Kommentar "first commit", der alle Änderungen im Ordner speichert
    • git branch -M main ⇒ Benennt den aktuellen Branch in main um (hieß früher Master)
    • git remote add origin https://github.com/MarzioCostantini/testgit.git ⇒ fügt einen neuen Remote Repository hinzu mit dem Namen "origin"
    • git push -u origin main ⇒ lädt alle lokalen Commits auf das Remote-Repository namens "origin" hoch und legt main als Standard Branch fest. Das "-u"-Argument gibt an, dass der lokale Branch beim nächsten Mal, wenn "git push" ausgeführt wird, automatisch mit dem Remote Repository synchronisiert wird.
  6. Wichtig: erst neuen Ordner erstellen und in VS öffnen
  7. in der Leiste oben über Terminal und dort neues Terminal öffnen
  8. Code bei diesem Ordner ins VS Terminal einfügen

Allgemeine Terminal-Befehle

  • git add . ⇒ fügt eine Änderung aus dem Arbeitsverzeichnis zur staging-area hinzu (Bereich, bevor es hochgeladen wird)
  • git add _FILENAME_ ⇒ fügt nur eine bestimmte Datei zur staging-area hinzu
  • git commit -m "Text" ⇒ erstellt einen neuen commit mit einer Nachricht, die kurz und auf Englisch erklärt, was geändert wurde
  • git push ⇒ lädt die Dateien aus der staging-area hoch
  • git log ⇒ zeigt Historie (sämtliche commits) meines aktuellen Repositorys
  • git log —oneline ⇒ zeigt nur die gemachten Änderungen, nicht Autor und Datum
  • git status ⇒ zeigt mir aktuelle Änderungen, die noch nicht hochgeladen sind im Repository (mit q kann ich die Ansicht verlassen)
  • git pull ⇒ lädt die geänderten Dateien herunter

Branches & Mergen

  • main branch ist mein Hauptzweig, den ich auch veröffentliche
  • daneben kann ich mehrere Nebenbranches aufmachen, auf denen ich zB Sachen ausprobiere, die nicht gleich veröffentlicht werden sollen oder zu Problemen führen können
  • erst wenn ich den Nebenbranch wieder mit dem Hauptbranch verbinde, werden die Änderungen des Nebenbranches in main übernommen und angezeigt
  • ich mache quasi eine Kopie des aktuellen Projekts, in der ich arbeite, und erst wenn die Kopie funktioniert, merge ich sie mit dem main branch
  • so besteht keine Gefahr für das eigentliche Projekt, das bisher ja gut funktionierte - wenn ich zB ein kompliziertes Burgermenü ergänzen will, kann ich einen Nebenbranch namens BurgerMenu öffnen und in diesem arbeiten

Terminal-Befehle für branches

  • in VSC sehe ich in der blauen Leiste unten links, auf welchem branch ich mich befinde
  • git branch ⇒ zeigt alle vorhandenen branches ist (* zeigt an, auf welchem branch man sich gerade befindet)
  • git branch Name ⇒ öffnet einen neuen branch, man bleibt aber auf dem main-branch
  • git checkout Name ⇒ wechselt auf einen anderen branch
  • git checkout -b Name ⇒ legt einen neuen branch an und wechselt direkt dorthin
  • git branch -d Name ⇒ löscht den branch wieder (sollte erst passieren, wenn der branch erfolgreich mit main gemergt wurde)

Terminal-Befehle fürs Mergen

  • git checkout main ⇒ in den branch navigieren, in den alles gemergt werden sollen, zB main
  • git merge Name ⇒ mergt den benannten branch in den branch, auf dem ich mich befinde
  • git add . / git commit -m “merge branch” / git push ⇒ gemergten branch pushen
  • git branch -d Name ⇒ löscht den gemergten branch (optional)

vorherigen Zustand wiederherstellen

  • git log bzw. git log —oneline ⇒ zeigt Historie an, in der ich die ID des Zustands herausfinden kann, den ich wiederherstellten will
  • git checkout ID . ⇒ geht zum Zustand dieser ID zurück und löscht alles, was dazwischen passiert ist
  • git add . / git commit -m “XYZ gelöscht” / git push ⇒ Änderungen pushen
